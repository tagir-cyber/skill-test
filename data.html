Работа с данными

В данном задании вам необходимо будет поработать с данными. У вас на входе будет массив данных в JSON формате, который необходимо будет обработать и на его основе сгенерировать разметку

После того как вы с помощью JSON сгенерируете разметку с карточками с продуктов, вам необходимо сделать по ним фильтрацию. Все необходимые данные, разметка и стили в проекте уже имеются

ТЗ

Шаг 1: Работа с данными

  1. Файл с данным находится по пути public/projects.json
  2. Файл с разметкой data.html
  3. Файлы со стилями source/scss/blocks/courses.scss и source/scss/blocks/product-card.scss
  4. Обертка/список в который необходимо сгенерировать карточки <ul class="courses__list" data-project="parent"></ul>. Внутри него должны сгенерироваться карточки
  5. Шаблон карточки находится в теге <template class="template-project-card">, который расположен после футтера.
  6. С помощью JS вам необходимо распарсить данные из JSON, вставить данные из массива в карточку шаблона и отрисовать ее внутри списка
  7. Структура JSON следующая JSON. Данные уже все есть и они валидны, их править не нужно
  8. После этого шага все данные должны быть обработаны и корректно отображены на страницы. Ожидаемый результат: Карточки генерируются на JS и изначально в размете этих карточек нет. Скриншот

Шаг 2: Фильтрация данных

  1. В разметке есть список <ul class="courses__filter-list"</ul> в котором находятся ссылки, элементы фильтра, означающие категорию фильтрации. Внутри атрибута href указан признак фильтрации. Точно такой же признак есть у каждой карточки в массиве JSON
  2. При нажатии на ссылку фильтра в списке ниже должны отображаться только те карточки, которые подходят по фильтру. К примеру мы нажали на ссылку "Игры" href которого равен #games. Соответственно ниже должна происходить фильтрация карточек, у которых есть в массиве tags (в JSON) соответствующий ключ. Скриншот
  3. Одновременно может быть выбрать только один фильтр
  4. При нажатии на кнопку фильтра, список должен отфильтроваться, а сама ссылка стать активной. За это отвечает класс на ссылке .is-active
  5. При выборе другого фильтра, активный класс с предыдущего должен сниматься
  6. При нажатии на ссылку "Все направления" фильтрация сбрасывается и показываются все элементы из данных
  7. Способ фильтрации вы можете выбрать сами. Можете скрывать лишние элементы либо удалять их из разметки. Мы рекомендуем при фильтрации очищать полностью список и генерировать новый, подходящий по фильтру.
  8. Весь код работы с данными и фильтрации нужно писать внутри папки source/script/modules/filter. Код должен быть разбит на несколько модулей. В одном файле не нужно писать всю логику, разбейте ее на смысловые блоки, функции и файлы. Итоговый файл / функцию нужно подключить к главному файлу source/script/main.js
  9. Не обязательно: Для удобства лучше писать логику с использованием классов.
  10. Не обязательно: При фильтрации в hash адреса страницы помещать выбранный фильтр. Т.е. фильтр изменился hash в адресной строки поменялся. И если сразу перейти на страницу с хэшом, то список будет сразу отфильтрован по необходимому признаку. Скриншот
  11. Не обязательно: Добавить микро анимацию при смене фильтров
  12. Пример работы фильтрации показан на видео
